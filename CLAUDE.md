# CLAUDE.md - 项目开发指南

## 沟通交互
- 使用**中文**与用户沟通
- 这是一个新项目，在编写模块前请详细确认需求，减少修改次数。

## 开发流程规范

### 1. 开发计划管理
- 所有开发计划详细记录在 `TODO.md` 文件中
- 使用复选框格式跟踪进度：
    - `- [ ]` 待完成
    - `- [x]` 已完成
- 每完成一项任务，**立即**在 `TODO.md` 中勾选对应项

### 2. 测试要求
- 每完成一次开发，**必须**进行测试
- 测试优先级：
    1. **优先自动化测试**：运行单元测试、集成测试
    2. **尝试手动验证**：如果可以通过命令行或脚本验证，自行执行
    3. **提示用户测试**：如果无法自行测试（如需要特定环境、API密钥、用户交互等），明确提示用户进行测试
- 只有测试通过后，才能在 `TODO.md` 中标记为完成状态

### 3. 状态标记规则
- ✅ 已完成并测试通过 → 勾选 `- [x]`
- 🔄 开发中 → 保持 `- [ ]`
- ⏳ 等待用户测试 → 添加备注说明

### 4. 需求确认
- 和用户确认后的项目需求应该记录在 `REQUIREMENT.md` 文件中，不仅仅是产品需求，还有对技术的需求。
